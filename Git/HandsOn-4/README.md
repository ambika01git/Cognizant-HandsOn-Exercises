# Git HandsOn-4

## Objective

Learn how to create and resolve merge conflicts in Git.

## Tasks Performed

- Verified the master branch was clean.
- Created a new branch `GitWork`.
- Added `hello.xml` in the branch.
- Committed changes to the branch.
- Switched to the master branch.
- Added a different version of `hello.xml`.
- Committed changes to the master branch.
- Compared the branch and master.
- Merged the branch into the master.
- Resolved the merge conflict manually.
- Updated `.gitignore` to ignore backup files.
- Deleted the merged branch.
- Verified the commit history and repository status.

## Files Included

- hello.xml
- .gitignore

## Result

Successfully created and resolved a merge conflict using Git and updated the repository.
